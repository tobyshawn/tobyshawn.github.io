远程接入

远程接入需要远程服务器开启了ssh服务，在本地使用终端或者工具进行远程连接

如SecureCRT、Xshell、FinalShell等等，以下以Xshell为例

1、新建连接，输入远程主机的IP地址

2、选择认证方式，如密码认证

3、如跳出以下情况，选择接受

4、成功连上远程主机

同样，Linux还可以通过终端软件对远程主机的文件进行操作，如xftp、finalshell等等

用户管理

基本概念

用户

既使用操作系统的人

  • 用户账号
    • 超级用户：用户名通常为root，UID一定为0
    • 系统用户：保留给系统使用，执行某些软件，系统内置或默认程序UID为1100；有系统账号需求的用户创建UID为101499
    • 普通用户：用户创建：用于登录的，UID为500~4294967295
  • 用户组：具有相同系统权限的一组用户
  • 组账号
    • 超级用户组：GID为0
    • 系统组：GID为1~499
    • 用户自定义组：GID为500~4294967295

账号文件

用户账号
用户名和uid等信息存放在/etc/passwd中
用户密码存放在/etc/shadow中
组账号
组名和gid等信息存放在/etc/group中
组密码存放在/etc/gshadow中

系统只识别uid和gid，用户名和组名只是便于记忆
用户名与uid、组名与gid都是一一对应（非人为修改情况下）

用户管理

添加用户usera、userb

[root@localhost /]# useradd usera
[root@localhost /]# useradd userb
  • -c comment 指定一段注释性描述。
  • -d 目录 指定用户主目录，如果此目录不存在，则同时使用-m选项，可以创建主目录。
  • -g 用户组 指定用户所属的用户组。
  • -G 用户组，用户组 指定用户所属的附加组。
  • -s Shell文件 指定用户的登录Shell。
  • -u 用户号 指定用户的用户号，如果同时有-o选项，则可以重复使用其他用户的标识号。

修改用户

[root@localhost /]# id usera
uid=1001(usera) gid=1001(usera) 组=1001(usera)
[root@localhost /]# id userb
uid=1002(userb) gid=1002(userb) 组=1002(userb)
[root@localhost /]# usermod -G userb usera
[root@localhost /]# id usera
uid=1001(usera) gid=1001(usera) 组=1001(usera),1002(userb)
[root@localhost /]# usermod -G usera userb
[root@localhost /]# id userb
uid=1002(userb) gid=1002(userb) 组=1002(userb),1001(usera)

删除用户

-r参数同时删除家目录

[root@localhost /]# userdel userb -r

切换用户

su [选项] 用户名
不带任何参数，普通用户切换到root用户，需要密码

密码管理

passwd

  • -l 锁定口令，即禁用账号。
  • -u 口令解锁。
  • -d 使账号无口令。
  • -f 强迫用户下次登录时修改口令。
  • 不加参数，修改密码
  • 高权限用户可指定修改用户密码

echo “password” | passwd –stdin username 此种方式也可以修改密码

用户信息文件

/etc/passwd

所有用户都可以对此文件执行读操作

[root@localhost /]# cat /etc/passwd
root:x:0:0:root:/root:/bin/bash
bin:x:1:1:bin:/bin:/sbin/nologin
daemon:x:2:2:daemon:/sbin:/sbin/nologin
adm:x:3:4:adm:/var/adm:/sbin/nologin
lp:x:4:7:lp:/var/spool/lpd:/sbin/nologin

账户名称：密码：用户ID：组ID：用户说明：账户家目录：登录的shell

  • 用户名：一串代表用户身份的字符串
  • 密码：x代表有设置密码，此处若为空代表登录时不需要密码，真正的密码保存在 /etc/shadow 文件中
  • uid：用户 ID，每个用户都有唯一的一个 UID，系统通过 UID 来识别不同的用户
  • gid：用户主要组的ID号
  • 描述性信息：记录用户全名与其它批注数据，这个字段并没有什么重要的用途，只是用来解释这个用户的意义而已
  • 家目录路径：用户登录后第一个要进入的目录
  • shell：用户登录后要执行程序的完整路径名称，默认为shell的/bin/bash

shadow影子文件

/etc/shadow

只有 root 用户拥有读权限，其他用户没有任何权限

[root@localhost /]# cat /etc/shadow
root:$6$VJOkPHVhMiE1Y6WK$eYorzGFixnfuTfeUiXnPfIIvUOGN3M6zWz3uBcghDLBlNBcSkFsZJHdpIwaY3XoeynVXsitOrpJma2hEN3XHp1::0:99999:7:::
bin:*:18353:0:99999:7:::
daemon:*:18353:0:99999:7:::
adm:*:18353:0:99999:7:::

登录名:加密口令:最后一次修改时间:最小时间间隔:最大时间间隔:警告时间(宽限):不活动时间:失效时间:标志

密码结构：$id$salt$encrypted；
id：加密方式；salt：随机字符串；encrypted：密文
id=1MD5；id=2aBlowfish；id=5SHA256；id=6SHA512

sudo配置

此命令让普通用户执行只有root才能执行的命令，使用root的uid和gid来执行命令

它的定义文件是/etc/sudoers，用命令visudo才能编辑sudoers文件

/etc/sudoers没有定义的账号，试图使用sudo执行命令，系统会通知管理员有未经允许的使用者试图执行未经允许的命令

sudo -l 查看当前用户可以以root身份运行的命令

/etc/sudoers

[root@localhost /]# cat /etc/sudoers | grep -v "^#" | grep "ALL="
root    ALL=(ALL)     ALL
%wheel    ALL=(ALL)    ALL

系统中的用户是否可以执行sudo命令，以及通过sudo可以执行哪些命令
用户/组 允许登录的主机=以什么身份执行 命令

用户组配置文件

/etc/group

[root@localhost /]# cat /etc/group | grep root
root:x:0:
[root@localhost /]# usermod -G root usera
[root@localhost /]# cat /etc/group | grep root
root:x:0:usera

组名:口令:组标识号:组内用户列表（初始用户不会显示在组用户中）

  • 组名：用户组的名称，由字母或数字构成
  • 组密码：x代表有设置密码，真正加密后的组密码默认保存在 /etc/gshadow 文件中
  • gid：组的ID号，系统是通过 GID 来区分用户组
  • 组成员：该组包含的所有用户

用户组gshadow影子文件配置文件

/etc/gshadow

[root@localhost /]# cat /etc/gshadow
root:::usera
bin:::
daemon:::
sys:::
adm:::
tty:::

用户组名:用户组密码:用户组管理员的名称:支持的账号名称

权限管理

权限介绍

-rw-r–r–

  • -文件类型（-文件 d目录 l软链接文件）
  • rw- u所有者
  • r– g所属组
  • r– o其他人

Linux下文件的权限类型包括读、写、执行，对应r、w、x，其权重分别为读权限（4）、写权限（2）、执行权限（1）、没有权限（0），即最高为777

权限对文件的作用：
r：可读取此文件的实际内容（cat more head tail）
w：可以编辑、新增或者修改文件的内容（vi echo），但是不包含删除文件
x：该文件具有可以被系统执行的权限（运行）

权限对目录的作用：
r：具有读取目录结构列表的权限，可以查询目录下文件名（ls）
w：具有修改目录结构的权限。如新建文件和目录，删除此目录下文件和目录，重命名此目录下文件和目录、剪切（touch rm mv cp）
x：代表用户能否进入该目录，可以进入目录（cd）

tips：对文件来讲，最高权限是执行；对目录来讲，最高权限是写

查看权限

可以使用 ll 或者 ls -l 命令来查看文件的权限

[root@localhost ~]# ll -sh
总用量 67M
4.0K -rw-------.  1 root root 1.4K 5月  31 20:29 anaconda-ks.cfg
4.0K drwxr-xr-x.  5 root root 4.0K 6月   7 11:37 cobaltstrike4.2-crack
3.9M -rw-r--r--.  1 root root 3.9M 6月   4 09:27 dircms.zip
   0 drwxr-xr-x.  2 root root   24 6月   3 10:27 test

Linux下权限粒度有所有者、所在组、其他组三种。通常一个文件只能归属于一个所有者和所在组，如果其他用户想要拥有这个文件的权限，就需要将该用户加入具备权限的群组，一个用户可以同时归属于多个组

[usera@localhost ~]$ ll
总用量 4
-rw-rw-r--. 1 usera usera 4 6月   7 16:23 123.txt
[usera@localhost ~]$ chmod +777 123.txt 
[usera@localhost ~]$ ll
总用量 4
-rwxrwxrwx. 1 usera usera 4 6月   7 16:23 123.txt

修改权限

权限修改

  • 使用数字修改文件权限
    chmod [选项] 权限值 文件名
    -R（注意是大写）选项表示连同子目录中的所有文件，也都修改设定的权限
    例如：chmod 777 file
  • 使用字母修改文件权限
    chmod [选项] 模式 文件名
    chmod u+x 1.txt
    chmod g+w,o+w 2.txt

所有者修改

  • 修改文件的所有者
    chown 用户名 文件名
    -例如：chown ds dengj.av
  • 修改文件的所属组
    chgrp 组名 文件名
    -例如：chgrp group1 fengj.av

可以使用umask 来查看默认权限
root用户默认是0022

  • 第一位0：文件特殊权限
  • 后三位022：文件默认权限

文件默认权限最大为666

文件（或目录）的初始权限 = 文件（或目录）的最大默认权限 - umask权限

特殊权限

特殊权限对执行文件的作用：

  • suid：使用者在执行这个带有suid特殊权限的执行文件时，是以文件拥有者在执行，而不是以执行者本身的权限来执行。（不要在目录上设置此权限）如：/usr/bin/passwd文件拥有suid权限，从而使得普通用户在执行此命令时会以root用户的权限去修改/etc/shadow的内容
  • sgid：使用者在执行这个带有sgid特殊权限的执行文件时，动态获取程序拥有者所在组的权限

特殊权限用字母表示

  • suid：u的最后x以s表示
  • sgid：g的最后x以s表示
  • sticky：o的最后x以t表示

suid的数字为4，sgid的数字为2，sticky的数字为1。放在第一位，即其它权限前面

设置特殊权限

  • chmod u+s file、chmod g+s file、chmod o+t file
  • chmod 2755 file 即g+s
  • chmod 7755 file 即u+s,g+s,o+t
