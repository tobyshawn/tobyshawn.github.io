网络

网卡配置

Centos 7

修改位置

/etc/sysconfig/network-scripts/ifcfg-xxx

内容解析

TYPE="Ethernet"                        #配置文件接口类型
DEVICE="eth0"                            #网络接口名称
BOOTPROTO="static"                #系统启动地址协议：static、dhcp、bootp、none
NM_CONTROLLED="no"                #是否由Network Manager控制该网络接口
ONBOOT="yes"                           #启动时是否激活
UUID=27137241-842e-4e50-88dd-8d8da1305dc0
DEFROUTE=yes                          #是否默认路由
IPADDR=192.168.1.1                #ip地址
NETMASK=255.255.255.0            #子网掩码
GATEWAY=192.168.1.254            #网关地址
BROADCAST=192.168.1.255        #广播地址
HWADDR=00:0C:29:13:5D:74    #MAC地址
#MACADDR=                                    #MAC地址，只需要设置一个，不能冲突
DNS1=114.114.114.114            #dns
DNS2=8.8.8.8                            #dns
HWADDR=00:90:27:50:5B:30
ARPCHECK=no

配置完成后

service network restart
#或systemctl restart network
#或reboot

Ubuntu 18.04之前

修改位置

/etc/network/interfaces

内容解析

auto lo
iface lo inet loopback

#static设置
auto eth0
iface eth0 inet static
     address 192.168.1.1                #IP地址
     netmask 255.255.255.0            #掩码
     broadcast 192.168.1.255        #广播地址
     gateway 192.168.1.254            #网关
#dhcp设置
auto eth1
iface eth1 inet dhcp

服务

Linux存在运行级别，可以理解为操作系统当前正在运行的程序的级别，程序的运行级别有以下0~6共7个运行级别

运行级别 效果
0 关机
1 单用户模式，只允许root登陆，作为维护使用
2 多用户，不完全的命令行模式，不支持网络功能（没NFS）
3 多用户，完全的命令行模式，带网络（有NFS）,默认进入文本模式
4 系统保留，但是默认和级别3相同
5 图形化界面，默认进入图形界面
6 重启

输入runlevel可以查看当前的运行级别

root@ubuntu:/# runlevel 
N 5

使用init 可以进入其他运行级别，如init 0为关机，init6为重启

在/etc文件夹下存在rc*.d的文件夹，系统会根据指定的运行级别进入对应的rc*.d目录，并按照文件名顺序检索目录下的链接文件

root@ubuntu:/# ls /etc/ | grep rc

rc1.d
rc2.d
rc3.d
rc4.d
rc5.d
rc6.d
rcS.d

软件管理

Linux的软件一般可以通过以下几种方式来安装

  • 源码
    • 需要通过编译成二进制文件才能进行使用，较为麻烦，但是兼容性好，可以在离线形式下安装
  • RPM
    • 将源代码基于特定平台系统进行编译成可执行文件，保持依赖关系
  • yum
    • 可以解决依赖关系，无需手动逐个安装RPM包
  • apt/apt-get
    • 类似yum，但是是debian，ubuntu发行版的管理工具

源码安装

  1. ./configure 定义需要的功能选项
    检测系统环境是否符合安装要求
    把定义好的功能选项和检测系统环境的信息都填写入makefile文件，用于后续的编辑
    ./configure –prefix=/usr/local/apache2 源码包一定要指定安装位置
  2. make 编译
    make命令会根据当前软件开发时的语言，自动识别跟调用相应的编译器，进行二进制编译
  3. make install 编译安装
    进行软件的安装（设置启动文件、移动执行文件）

卸载时直接删除安装目录即可，不会遗留任何垃圾文件

  • 优点：
    开源，如果有足够的能力，可以修改源代码
可以自由选择所需的功能
软件是编译安装，所以更加适合自己的系统，更加稳定、效率更高
卸载方便（安装目录直接删掉，没有任何垃圾文件）
  • 缺点：
    安装过程步骤较多，尤其安装较大的软件集合时（如LAMP环境搭建）
编译过程时间较长
因为是编译安装，安装过程中一旦报错新手很难解决

RPM包安装

命名规则：name-version-release.arch.rpm

httpd-2.2.15-15.el6.centos.1.i686.rpm

  • httpd:软件包名
  • 2.2.15:软件版本
  • 15:软件发布的次数
  • el6:软件发行商，el6表示此包是由redhat公司发布
  • centos:适合的linux平台
  • i686:适合的硬件平台
  • rpm:rpm包扩展名

安装命令

rpm -ivh 包全名
选项：
-i 安装
-v 显示详细信息
-h 显示安装进度
–nodeps 不检测依赖性

升级命令

rpm -Uvh 包全名

卸载命令

rpm -e 包名
选项：
-e 卸载
–nodeps 不检查依赖性

YUM

查询

yum list #查询所有可用软件包列表
yum search 关键字 #搜索服务器上所有和关键字相关的包
例：yum search httpd

安装

yum -y install 包名
选项：
install 安装
-y 自动回答yes
例如：yum -y install gcc
yum安装只写包名即可。

升级

yum -y update 包名
选项：
-update 升级
-y 自动回答yes

卸载

yum -y remove 包名
选项：
-remove 卸载
-y 自动回答yes

镜像源

需在root权限下

Centos 7

#1、做好备份，为了更新失败时切换回去
mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup
#2、进入yum源配置文件夹
cd /etc/yum.repos.d/
#3、下载对应的新源，以下任选一句
wget -O /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo 
curl -O http://mirrors.aliyun.com/repo/Centos-7.repo 
#4执行更新
yum clean all
yum makecache
yum -y update

Ubuntu 18.04

#1、备份原文件
cp /etc/apt/sources.list /etc/apt/sources.list.bak
#2、新增源文件
vim /etc/apt/sources.list
#3、输入以下内容，可根据版本在网上找到合适的源
#4、执行更新
apt-get update
apt-get upgrade
